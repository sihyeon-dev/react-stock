# 심플 주식 대시보드 기획서

| 구분 | 내용 |
|---|---|
| 프로젝트명 | 심플 주식 대시보드 |
| 개발 형태 | React 기반 웹 애플리케이션 |
| 사용 API | Alpha Vantage API |
| 주요 기능 | 미국 주식 티커 검색, 주가 정보 조회, 1개월 종가 차트, 관련 뉴스, 관심종목 저장 |
| 개발 목적 | React 수업에서 배운 내용을 실제 프로젝트에 적용하기 위해 기획했습니다. |

---

## 1. 프로젝트명

프로젝트명은 심플 주식 대시보드입니다.

이 프로젝트는 사용자가 미국 주식 티커를 입력하면 해당 종목의 현재 주가 정보, 최근 1개월 종가 흐름, 관련 뉴스, 관심종목 목록을 확인할 수 있는 React 기반 웹 애플리케이션입니다.

주식 정보 사이트나 증권 앱처럼 많은 정보를 한 번에 보여주기보다는, 사용자가 처음 주식을 접할 때 필요한 핵심 정보 위주로 간단하게 확인할 수 있도록 구성했습니다.

---

## 2. 프로젝트 주제

본 프로젝트는 Alpha Vantage API를 활용해 미국 주식 정보를 조회하는 미니 대시보드입니다.

사용자가 `AAPL`, `MSFT`, `NVDA`, `TSLA`, `AMD` 같은 미국 주식 티커를 입력하면 현재가, 등락폭, 등락률, 오늘 고가, 오늘 저가, 오늘 시가, 전일 종가를 화면에 표시합니다.

또한 최근 일별 종가 데이터를 이용해 1개월 종가 차트를 SVG로 보여주고, 검색한 종목과 관련된 뉴스도 함께 제공합니다. 사용자는 검색한 종목을 관심종목으로 저장할 수 있으며, 저장한 종목을 다시 조회하거나 삭제할 수도 있습니다.

이 프로젝트는 단순히 주식 정보를 보여주는 것뿐만 아니라, React에서 상태 관리와 컴포넌트 분리, API 요청 흐름을 연습하는 데 중점을 두었습니다.

---

## 3. 기획 배경 및 목적

기존 주식 정보 사이트나 증권 앱은 제공하는 정보가 많아 초보자가 보기에는 복잡하게 느껴질 수 있습니다. 특히 처음 주식을 공부하는 사람은 어떤 정보가 중요한지 구분하기 어렵고, 화면에 표시되는 수많은 지표 때문에 오히려 정보를 이해하기 힘들 수 있습니다.

그래서 이 프로젝트에서는 사용자가 필요한 핵심 주식 정보만 간단하게 확인할 수 있는 대시보드를 만들고자 했습니다. 사용자가 미국 주식 티커만 입력하면 기본 가격 정보, 최근 가격 흐름, 관련 뉴스를 한 화면에서 볼 수 있도록 구성했습니다.

또한 이 프로젝트는 React 수업에서 배운 내용을 실제 프로젝트에 적용하는 것을 목적으로 합니다. 컴포넌트 분리, props 전달, `useState`, `useEffect`, 조건부 렌더링, 배열 렌더링, API 요청, 로딩 처리, 에러 처리, `localStorage` 저장 방식을 하나의 프로젝트 안에서 연습할 수 있도록 만들었습니다.

이를 통해 단순한 예제 코드가 아니라, 실제 데이터를 가져와 화면에 출력하는 웹 애플리케이션의 기본 구조를 이해하는 것을 목표로 했습니다.

## 4. 사용 API

이 프로젝트에서는 Alpha Vantage API를 사용합니다. Alpha Vantage는 주식 가격 정보, 시계열 데이터, 뉴스 데이터를 제공하는 API입니다.

API 기본 주소는 다음과 같습니다.

```txt
https://www.alphavantage.co/query
```

사용한 API 기능은 다음과 같습니다.

| API 기능 | 사용 목적 |
|---|---|
| `GLOBAL_QUOTE` | 현재가, 등락폭, 등락률, 오늘 고가, 오늘 저가, 오늘 시가, 전일 종가를 조회합니다. |
| `TIME_SERIES_DAILY` | 최근 일별 종가 데이터를 가져와 1개월 종가 차트를 만듭니다. |
| `NEWS_SENTIMENT` | 검색한 종목과 관련된 뉴스 제목, 출처, 요약, 감성 라벨을 가져옵니다. |

API Key는 `.env` 파일에 저장합니다.

```txt
VITE_ALPHA_VANTAGE_API_KEY=발급받은_API_KEY
```

React 코드에서는 다음 환경변수로 API Key를 사용합니다.

```js
import.meta.env.VITE_ALPHA_VANTAGE_API_KEY
```

프론트엔드에서 사용하는 `.env`는 코드 관리에는 도움이 되지만 완전한 보안 방법은 아닙니다. 실제 서비스로 확장한다면 백엔드 서버에서 API Key를 관리하는 방식이 더 적절합니다. 이 프로젝트에서는 수업용 미니 프로젝트이기 때문에 `.env` 파일을 이용해 API Key를 관리했습니다.

## 5. 화면 구성

화면은 크게 상단 제목 영역, 탭 메뉴, 종목 검색 화면, 관심종목 화면으로 구성했습니다.

### 5.1 상단 영역

상단 영역에는 앱 제목과 간단한 설명 문구를 표시합니다.

| 항목 | 내용 |
|---|---|
| 제목 | 심플 주식 대시보드 |
| 설명 | 미국 주식 티커를 검색하고 관심종목을 저장해보세요. |

상단 영역을 통해 사용자가 이 앱이 어떤 기능을 제공하는지 바로 이해할 수 있도록 했습니다.

### 5.2 탭 메뉴

탭 메뉴는 두 개로 구성했습니다.

| 탭 | 역할 |
|---|---|
| 종목 검색 | 티커 검색, 주가 정보, 차트, 뉴스 결과를 보여줍니다. |
| 관심종목 | 사용자가 저장한 관심종목 목록을 보여줍니다. |

관심종목 탭에는 현재 저장된 관심종목 개수도 함께 표시합니다. 이를 통해 사용자는 자신이 몇 개의 종목을 저장했는지 쉽게 확인할 수 있습니다.

### 5.3 종목 검색 화면

종목 검색 화면에는 티커 입력창, 검색 버튼, 빠른 선택 버튼, 에러 메시지, 로딩 메시지, 검색 결과 카드가 있습니다.

사용자는 입력창에 원하는 미국 주식 티커를 입력하고 검색 버튼을 눌러 정보를 조회할 수 있습니다. 예를 들어 `AAPL`을 입력하면 애플의 주식 정보를 조회할 수 있습니다.

빠른 선택 버튼에는 대표 종목인 `AAPL`, `MSFT`, `NVDA`, `TSLA`, `AMD`를 배치했습니다. 사용자는 직접 티커를 입력하지 않아도 버튼을 눌러 바로 검색할 수 있습니다.

검색 중일 때는 로딩 문구를 보여주고, 잘못된 티커를 입력하거나 API 요청에 문제가 생기면 에러 메시지를 화면에 표시합니다.

### 5.4 검색 결과 카드

검색 결과 카드에는 종목 정보와 뉴스가 함께 표시됩니다.

왼쪽 영역에는 현재가, 등락폭, 등락률, 오늘 고가, 오늘 저가, 오늘 시가, 전일 종가, 1개월 종가 차트가 표시됩니다. 이 정보들을 통해 사용자는 해당 종목의 기본적인 가격 상황을 빠르게 확인할 수 있습니다.

오른쪽 영역에는 관련 뉴스 목록이 표시됩니다. 뉴스 제목을 클릭하면 원문 링크로 이동할 수 있도록 구성했습니다. 뉴스에는 제목, 출처, 날짜, 감성 라벨, 요약 정보가 포함됩니다.

관심종목 버튼은 검색 결과 카드의 오른쪽 위에 배치했습니다. 저장되지 않은 종목에서는 `관심종목 추가`로 표시되고, 이미 저장된 종목에서는 `관심종목 해제`로 표시됩니다. 버튼을 다시 누르면 관심종목에서 제거되는 토글 방식으로 동작합니다.

### 5.5 관심종목 화면

관심종목 화면에는 사용자가 저장한 티커 목록을 보여줍니다.

각 종목에는 다시 검색하는 조회 버튼과 목록에서 제거하는 삭제 버튼이 있습니다. 조회 버튼을 누르면 해당 종목의 정보를 다시 검색할 수 있고, 삭제 버튼을 누르면 관심종목 목록에서 제거됩니다.

관심종목이 없을 때는 `아직 관심종목이 없습니다.`라는 안내 문구를 표시합니다. 이를 통해 사용자가 현재 저장된 종목이 없다는 것을 쉽게 알 수 있도록 했습니다.

## 6. 데이터 흐름

사용자가 티커를 입력하고 검색 버튼을 누르면 먼저 입력값의 앞뒤 공백을 제거하고 대문자로 변환합니다. 예를 들어 사용자가 `aapl`을 입력하면 `AAPL`로 바꾸어 API 요청에 사용합니다.

그 다음 `fetchQuote()` 함수를 실행하여 Alpha Vantage의 `GLOBAL_QUOTE` API에 현재 주식 정보를 요청합니다. 요청이 성공하면 받아온 데이터를 `stockData` 상태에 저장하고, 검색한 티커를 `selectedSymbol` 상태에 저장합니다.

현재 주식 정보 요청이 성공하면 `fetchMonthlyCandles()`와 `fetchCompanyNews()`를 순서대로 실행합니다. 코드 흐름을 쉽게 이해할 수 있도록 `await`를 사용해 필요한 API를 차례대로 요청합니다.

차트 데이터는 `chartData` 상태에 저장합니다. 뉴스 데이터는 `stockNews` 상태에 저장합니다. API 요청 중 문제가 생기면 `error` 상태에 안내 메시지를 저장하고 이전 검색 결과를 비웁니다.

상태값이 변경되면 React가 화면을 다시 렌더링하여 종목 정보, SVG 차트, 뉴스 목록을 표시합니다.

관심종목을 추가하거나 해제하면 `watchlist` 상태가 변경됩니다. `watchlist`가 변경될 때마다 `useEffect`가 실행되고, 이 값을 `localStorage`에 저장합니다. 앱이 처음 실행될 때는 `localStorage`에서 저장된 관심종목을 불러와 화면에 표시합니다.

이러한 흐름을 통해 사용자가 검색한 데이터가 상태에 저장되고, 저장된 상태값을 기준으로 화면이 자동으로 바뀌게 됩니다.

## 7. 컴포넌트 구조

현재 프로젝트의 컴포넌트 구조는 검색, 종목정보, 뉴스, 관심종목 네 가지 기능 중심으로 단순하게 나누었습니다.

```txt
src/
  App.jsx
  main.jsx
  api/
    stockApi.js
  components/
    StockSearch.jsx
    StockInfo.jsx
    StockNews.jsx
    Watchlist.jsx
  styles/
    App.css
```

컴포넌트를 나눈 이유는 한 파일에 모든 코드를 작성하면 코드가 길어지고 관리하기 어려워지기 때문입니다. 기능별로 파일을 나누면 각 컴포넌트가 어떤 역할을 하는지 쉽게 파악할 수 있고, 수정할 때도 필요한 부분만 찾아서 고칠 수 있습니다.

### 7.1 App.jsx

`App.jsx`는 프로젝트의 중심 컴포넌트입니다. 전체 상태를 관리하고, API 요청 함수를 실행하며, 검색 탭과 관심종목 탭을 연결합니다.

`App.jsx`에서 관리하는 주요 상태는 다음과 같습니다.

| 상태 이름 | 역할 |
|---|---|
| `symbol` | 검색창에 입력되는 티커를 저장합니다. |
| `selectedSymbol` | 실제로 조회에 성공한 티커를 저장합니다. |
| `stockData` | 검색한 종목의 현재 주가 정보를 저장합니다. |
| `chartData` | 1개월 차트에 사용할 종가 데이터를 저장합니다. |
| `stockNews` | 관련 뉴스 목록을 저장합니다. |
| `watchlist` | 관심종목 목록을 저장합니다. |
| `loading` | API 요청 중인지 표시합니다. |
| `error` | 검색 또는 API 에러 메시지를 저장합니다. |
| `activeTab` | 현재 선택된 탭을 저장합니다. |

사용자가 종목을 검색하면 `App.jsx`에서 API 요청을 실행하고, 받아온 데이터를 각각의 상태에 저장합니다. 저장된 데이터는 props를 통해 하위 컴포넌트로 전달합니다.

예를 들어 검색창의 입력값은 `symbol` 상태로 관리하고, 검색에 성공한 종목 정보는 `stockData`에 저장합니다. 이후 `stockData`, `chartData`, `stockNews`를 `StockInfo`와 `StockNews` 컴포넌트에 전달하여 화면에 출력합니다.

또한 관심종목 추가, 해제, 삭제, 조회 기능도 `App.jsx`에서 관리합니다. 관심종목이 변경되면 `useEffect`를 이용해 `localStorage`에 저장하여 새로고침 후에도 데이터가 유지되도록 했습니다.

### 7.2 src/api/stockApi.js

`stockApi.js`는 API 요청 코드를 모아두는 파일입니다. 화면을 담당하는 컴포넌트와 API 요청 코드를 분리하여 `App.jsx`가 더 깔끔해지도록 구성했습니다.

이 파일에는 다음 함수가 있습니다.

| 함수 이름 | 역할 |
|---|---|
| `fetchQuote(symbol)` | 입력한 티커의 현재 주가 정보를 요청합니다. |
| `fetchMonthlyCandles(symbol)` | 최근 일별 종가 데이터를 요청하고 차트 데이터 형태로 변환합니다. |
| `fetchCompanyNews(symbol)` | 입력한 티커와 관련된 뉴스 목록을 요청합니다. |

API 요청 함수를 따로 분리하면 나중에 API 주소나 요청 방식이 바뀌어도 `stockApi.js` 파일만 수정하면 됩니다. 또한 `App.jsx`에서는 필요한 함수만 가져와 사용할 수 있기 때문에 코드가 더 정리되어 보입니다.

### 7.3 StockSearch.jsx

`StockSearch.jsx`는 검색 컴포넌트입니다.

이 컴포넌트는 티커 입력창, 검색 버튼, 빠른 선택 버튼을 담당합니다. 사용자가 검색창에 티커를 입력하면 입력값이 `symbol` 상태에 저장됩니다.

`StockSearch.jsx`는 직접 API를 요청하지 않고, 사용자의 입력과 클릭 이벤트를 `App.jsx`로 전달합니다. 예를 들어 사용자가 검색 버튼을 누르면 `onSearch()` 함수가 실행되고, 빠른 선택 버튼을 누르면 해당 티커가 `App.jsx`로 전달됩니다.

즉, `StockSearch.jsx`는 사용자의 입력을 받는 역할을 하고, 실제 API 요청과 데이터 관리는 `App.jsx`가 담당합니다. 이렇게 역할을 나누면 검색 화면 코드와 데이터 처리 코드를 분리할 수 있습니다.

### 7.4 StockInfo.jsx

`StockInfo.jsx`는 종목정보 컴포넌트입니다.

이 컴포넌트는 현재가, 등락폭, 등락률, 오늘 고가, 오늘 저가, 오늘 시가, 전일 종가를 카드 형태로 보여줍니다. 등락률이 양수이면 초록색, 음수이면 빨간색, 0이면 회색으로 표시하여 사용자가 상승과 하락을 쉽게 구분할 수 있도록 했습니다.

또한 `chartData`를 이용해 최근 1개월 종가 흐름을 SVG `polyline` 차트로 표시합니다. 차트에는 데이터 개수, 최저 종가, 최고 종가도 함께 표시합니다.

관심종목 토글 버튼도 이 컴포넌트에 있으며, 검색 결과 카드의 오른쪽 위에 배치했습니다. 사용자가 현재 보고 있는 종목을 바로 관심종목에 추가하거나 해제할 수 있도록 했습니다.

### 7.5 StockNews.jsx

`StockNews.jsx`는 뉴스 컴포넌트입니다.

이 컴포넌트는 Alpha Vantage의 `NEWS_SENTIMENT` API에서 가져온 관련 뉴스 목록을 표시합니다. 뉴스 제목, 출처, 날짜, 감성 라벨, 요약을 보여주며, 뉴스 제목을 클릭하면 원문 링크로 이동할 수 있습니다.

뉴스 데이터가 없을 때는 관련 뉴스가 없다는 안내 문구를 표시합니다. 이를 통해 뉴스 데이터가 없더라도 화면이 비어 보이지 않도록 했습니다.

### 7.6 Watchlist.jsx

`Watchlist.jsx`는 관심종목 컴포넌트입니다.

사용자가 저장한 티커 목록을 보여주고, 각 티커마다 조회 버튼과 삭제 버튼을 제공합니다. 조회 버튼을 누르면 해당 티커를 다시 검색하고, 삭제 버튼을 누르면 관심종목 목록에서 제거합니다.

관심종목 데이터는 `App.jsx`의 `watchlist` 상태에서 관리하고, `Watchlist.jsx`는 이 값을 props로 전달받아 화면에 출력합니다.

관심종목이 없을 때는 안내 문구를 표시하여 사용자가 현재 저장된 종목이 없다는 것을 알 수 있도록 했습니다.

### 7.7 App.css

`App.css`는 전체 화면 디자인을 담당합니다.

카드 레이아웃, 버튼 디자인, 입력창 스타일, 상승/하락 색상, 뉴스 목록, 관심종목 목록, 반응형 화면을 스타일링합니다.

특히 주가 등락률에 따라 색상을 다르게 표시하고, 카드 형태로 정보를 정리하여 사용자가 화면을 보기 쉽게 만들었습니다. 또한 화면 크기가 줄어들어도 사용할 수 있도록 반응형 스타일도 적용했습니다.

## 11. 느낀점

이번 프로젝트를 진행하면서 API를 사용하는 웹 애플리케이션은 단순히 데이터를 가져와 출력하는 것뿐만 아니라, 로딩 상태와 에러 상황을 함께 고려해야 한다는 점을 알게 되었습니다.

처음에는 API에서 데이터를 받아오면 바로 화면에 보여주면 된다고 생각했지만, 실제로는 요청이 진행 중일 때 사용자에게 알려주는 로딩 처리와 요청에 실패했을 때 보여주는 에러 처리가 필요했습니다. 현재 프로젝트에서는 코드 흐름을 쉽게 이해할 수 있도록 API 요청을 순서대로 실행하고, 문제가 생기면 하나의 에러 메시지로 안내하도록 구성했습니다.

React에서는 하나의 파일에 모든 코드를 작성하는 것보다 검색, 종목정보, 뉴스, 관심종목처럼 기능별로 컴포넌트를 나누는 것이 더 이해하기 쉽다는 것을 알 수 있었습니다. 컴포넌트를 분리하니 각 파일이 담당하는 역할이 명확해졌고, 코드를 수정할 때도 필요한 부분을 찾기 쉬웠습니다.

특히 `useState`를 이용해 화면 상태를 관리하고, `useEffect`를 이용해 관심종목을 저장하고 불러오는 과정을 통해 React의 상태 변화와 렌더링 흐름을 더 잘 이해할 수 있었습니다. 이번 프로젝트를 통해 수업에서 배운 React 문법이 실제 웹 애플리케이션에서 어떻게 사용되는지 경험할 수 있었습니다.
